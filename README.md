## What It Is

FinView is a full-stack financial dashboard that combines live market data with AI-powered analysis across six tabs. It started as a personal tool and became a production-grade engineering showcase covering SSE infrastructure, prompt engineering for deterministic AI output, real-time data feeds, and per-user budget controls.

Built and deployed to production, actively used.

## Six Tabs

### Watchlist
Live stock quotes for a configurable ticker list. Displays real-time price, change, and percentage. Extended-hours prices (pre-market and after-hours) are fetched via a multi-source waterfall: Pyth Network as primary, Google Finance as fallback, yfinance for supplemental coverage -- handling gaps across tickers and trading sessions.

### Market Snapshot
AI-generated summary of current market conditions across major indices, sectors, and macro themes. Claude synthesizes live data into a concise, readable read on where the market stands right now.

### Market Outlook
Forward-looking AI analysis: what the data suggests about near-term direction, key levels to watch, and macro tailwinds or headwinds. Distinct from Snapshot -- this tab is explicitly about what may be coming, not what is.

### Hype Detector
The most technically interesting tab. Claude returns a custom HTML dashboard -- ticker cards with sentiment scores, hype indicators, and visual heat signals. The challenge: Claude's output was non-deterministic. Depending on phrasing, it would return raw HTML sometimes and structured prose other times, making reliable rendering impossible.

The fix was prompt engineering: explicit output contracts, format anchors, and negative examples baked into the system prompt. Output is now deterministic across runs. See [docs/PROMPT_ENGINEERING.md](./docs/PROMPT_ENGINEERING.md) for the full breakdown.

### World Events
AI analysis of macro geopolitical and economic events and their potential market impact. Claude maps global headlines to sector exposure and portfolio-level implications.

### Catalyst Calendar
**No AI -- intentional.** Originally built with AI-generated catalyst data. After evaluating output quality, AI was removed entirely and the tab was rebuilt as a pure data feed pulling from Yahoo Finance (earnings dates), SEC EDGAR (8-K filings), SEC Form 4 (insider transactions), and live news headlines.

The decision: AI-generated financial calendars introduced hallucinated dates and fabricated events that looked identical to real ones. A feed that is always accurate is more valuable than an AI output that is sometimes impressive. Knowing when not to use AI is part of the engineering.

## Engineering Highlights

**SSE for live cache streaming**
Server-Sent Events push cache refresh status to the browser in real time. Users see data updating as it happens -- no polling, no page reloads, no stale reads.

**Extended-hours price feeds**
Pre-market and after-hours prices are not available from standard quote APIs at low cost. FinView uses a multi-source waterfall: Pyth Network (real-time decentralized price feeds) primary, Google Finance fallback, yfinance supplemental. Each source covers different tickers and sessions.

**Deterministic Claude output via prompt engineering**
The Hype Detector tab renders a custom HTML dashboard generated by Claude. Early builds produced inconsistent output -- sometimes HTML, sometimes prose, depending on subtle phrasing differences. The solution was a structured prompt contract: explicit format specification, hard output boundaries, and negative examples showing what not to return. Output is now consistent across runs. Full case study in [docs/PROMPT_ENGINEERING.md](./docs/PROMPT_ENGINEERING.md).

**DB-backed prompt management**
All AI prompts are stored in PostgreSQL and loaded at runtime -- not hardcoded. Prompts can be updated, versioned, and rolled back without a redeploy. Each tab's prompt is managed independently.

**Per-user AI usage tracking and budget controls**
Superadmin dashboard tracks Claude API usage per user: token counts, cost estimates, and configurable budget limits. Usage is logged per request. Admins can set per-user caps to prevent runaway spend.

**Catalyst tab: intentional AI removal**
Evaluated AI-generated catalyst data, determined output quality did not meet the bar for financial data reliability, and rebuilt as a pure data tab. The discipline to remove AI when it does not belong is as important as knowing how to apply it.

## Why I Built This

My husband was looking for a way to consolidate all the different views he checks throughout the day for stock research -- prices, extended-hours data, market analysis, world events, all in one place. He came up with the tabs and the requirements. We built it together, engineering each feature using AI and data sources including Anthropic Claude, Pyth Network, Yahoo Finance, SEC EDGAR, FRED, and Finnhub. The collaboration pushed the engineering in good directions: SSE infrastructure, deterministic AI output via prompt engineering, real-time data waterfall design, and the discipline to pull AI out of a tab when the output quality did not justify the risk.

The Catalyst decision is my favorite part. The AI-generated dates were wrong, causing Claude to hallucinate catalyst events that looked real but weren't. We pulled it out and rebuilt as a pure data feed from Yahoo Finance, SEC EDGAR, and live news. Knowing when not to use AI matters as much as knowing how to use it.
